## 🎯 接口设计要点

### 重要说明

1. **module_id 必须对应已存在的模块Id**
2. **id 字段需要提供**，使用 uuid
3. **method 必须是有效值**：get、post、put、delete、patch

### 接口设计规范

请根据系统模块设计接口文档，需要覆盖以下方面：

1. **RESTful 规范**
   - 使用标准的 HTTP 方法
   - 接口路径命名规范
   - 版本控制（如 /api/v1/xxx）

2. **接口定义**
   - 接口名称（name）：清晰表达接口功能
   - 接口路径（path）：完整的 URL 路径
   - 请求方法（method）：get/post/put/delete/patch
   - 接口描述（description）：接口功能说明

3. **请求参数设计**
   - 请求头（request_headers）：如 Authorization
   - 路径参数（request_params）
   - 请求体（request_body）：POST/PUT/PATCH
   - 参数类型：string、number、object、array
   - 是否必填
   - 参数说明

4. **响应格式设计**
   - 成功响应格式
   - 错误响应格式
   - 错误码定义

5. **模块归属**
   - 每个接口必须归属到对应的模块
   - module_id 必须与系统模块对应
