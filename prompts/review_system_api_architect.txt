## 🎯 架构师评审要点

请重点关注以下方面：

### 1. 与架构一致性
- 接口设计是否符合系统架构
- 模块间接口划分是否合理
- 是否有不合理的跨模块调用

### 2. 模块归属正确性
- 接口的 module_id 是否正确
- 是否存在归属错误的接口
- 模块接口划分是否合理

### 3. 接口粒度
- 接口粒度是否合理
- 是否存在过胖或过瘦接口
- 是否便于服务拆分

### 4. 技术合理性
- HTTP 方法选择是否恰当
- 路径设计是否规范
- 版本控制是否合理

### 5. 扩展性
- 接口是否便于扩展
- 是否有足够的扩展预留
- 是否支持未来功能演进

## 🔄 评审通过条件

**重要**：只有当 `system_api_issues` 为空数组时，评审才算通过。

### 如果有 system_api_issues：
- 返回具体的修改建议
- 流程将返回给后端修改，修改后再次评审

### 如果 system_api_issues 为空：
- 评审通过
