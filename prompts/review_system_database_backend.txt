## 🎯 后端评审要点

请重点关注以下方面：

### 1. ORM 映射
- 表结构是否便于 ORM 映射
- 字段命名是否符合编程规范
- 数据类型是否便于代码处理
- 是否有合理的默认值

### 2. 事务处理
- 事务边界是否清晰
- 是否支持业务事务需求
- 并发控制设计
- 锁粒度设计

### 3. 数据访问层
- 是否便于数据访问层封装
- 批量操作支持
- 分页查询支持
- 联表查询复杂度

### 4. API 数据返回
- 是否便于接口数据组装
- 关联查询是否高效
- 是否有 N+1 查询风险

### 5. 业务实现友好
- 字段是否便于业务逻辑处理
- 枚举值设计
- 状态字段设计
- 时间字段设计

## 🔄 评审通过条件

**重要**：只有当 `system_database_issues` 为空数组时，评审才算通过。

### 如果有 system_database_issues：
- 返回具体的修改建议
- 流程将返回给 DBA 修改，修改后再次评审

### 如果 system_database_issues 为空：
- 评审通过
