## 🎯 运维工程师评审要点

请重点关注以下方面：

### 1. 备份策略
- 备份方案是否完善
- 备份频率是否合理
- 恢复时间目标（RTO）是否满足
- 数据恢复目标（RPO）是否满足

### 2. 容量规划
- 数据增长预估
- 存储容量规划
- 性能容量规划
- 扩容方案

### 3. 数据迁移
- 数据库版本兼容性
- 数据迁移方案
- 灰度发布支持
- 回滚方案

### 4. 高可用设计
- 主从复制方案
- 读写分离方案
- 故障切换方案
- 数据同步机制

### 5. 监控运维
- 监控指标是否完善
- 告警阈值设置
- 慢查询监控
- 连接池管理

### 6. 安全运维
- 数据库账号权限管理
- 敏感数据访问审计
- 网络隔离方案
- 漏洞修复策略

## 🔄 评审通过条件

**重要**：只有当 `system_database_issues` 为空数组时，评审才算通过。

### 如果有 system_database_issues：
- 返回具体的修改建议
- 流程将返回给 DBA 修改，修改后再次评审

### 如果 system_database_issues 为空：
- 评审通过

## 📤 输出要求

完成评审后，请调用 `review_system_database_output` 工具输出结果：

```json
{
  "system_database_issues": [
    // 只有发现问题才填写，如果数据库已经完善则为空数组[]

      "content": "针对数据库提出的具体问题描述",
      "propose": "建议的解决方案"
    }
  ]
}
```

## ⚠️ 注意事项

- `system_database_issues` 为空 → 评审通过
- `system_database_issues` 有内容 → 需要 DBA 修改，修改后再评审
